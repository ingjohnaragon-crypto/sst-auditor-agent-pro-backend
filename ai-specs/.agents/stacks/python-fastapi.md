## Domain Layer Rules

### Models
- Pure Python dataclasses or `@dataclass` classes — zero framework dependencies
- Business logic lives inside domain methods
- Use `__init__` or classmethods as factory methods to enforce invariants

```python
from dataclasses import dataclass

@dataclass
class Candidate:
    id: int | None
    first_name: str
    email: str

    @classmethod
    def create(cls, first_name: str, email: str) -> "Candidate":
        if "@" not in email:
            raise InvalidEmailError(email)
        return cls(id=None, first_name=first_name, email=email)
```

### Repository Interfaces
- Abstract base classes in the domain layer
- Implementations live in infrastructure
- Return domain models, not SQLAlchemy ORM objects

```python
from abc import ABC, abstractmethod

class CandidateRepository(ABC):
    @abstractmethod
    async def find_by_id(self, id: int) -> Candidate | None: ...

    @abstractmethod
    async def save(self, candidate: Candidate) -> Candidate: ...

    @abstractmethod
    async def exists_by_email(self, email: str) -> bool: ...
```

## Application Layer Rules

- Services receive repository interfaces via constructor injection
- Accept Pydantic DTOs — never raw `dict` from request
- Raise domain exceptions for business rule violations
- No SQLAlchemy or FastAPI imports in this layer

```python
class CandidateService:
    def __init__(self, repo: CandidateRepository) -> None:
        self._repo = repo

    async def create(self, dto: CreateCandidateRequest) -> CandidateResponse:
        if await self._repo.exists_by_email(dto.email):
            raise DuplicateEmailError(dto.email)
        candidate = Candidate.create(dto.first_name, dto.email)
        saved = await self._repo.save(candidate)
        return CandidateMapper.to_response(saved)
```

## Presentation Layer Rules

- Routers are thin — validate via Pydantic, call service, return response
- Use FastAPI `Depends()` for dependency injection of services
- HTTP status codes: 201 for POST create, 200 for GET/PUT, 204 for DELETE
- Register global exception handlers on the FastAPI `app` instance

```python
router = APIRouter(prefix="/candidates", tags=["Candidates"])

@router.post("/", status_code=201, response_model=CandidateResponse)
async def create_candidate(
    request: CreateCandidateRequest,
    service: CandidateService = Depends(get_candidate_service),
) -> CandidateResponse:
    return await service.create(request)
```

### Global Exception Handlers
```python
@app.exception_handler(DomainException)
async def domain_exception_handler(request: Request, exc: DomainException):
    return JSONResponse(status_code=exc.http_status,
        content={"success": False, "code": exc.code, "message": str(exc)})

@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(status_code=422,
        content={"success": False, "code": "VALIDATION_ERROR", "details": exc.errors()})
```

## Infrastructure Layer Rules

### Migrations (Alembic)
- All schema changes go through Alembic — never use `Base.metadata.create_all()` in production
- `alembic revision --autogenerate -m "description"` to create
- `alembic upgrade head` to apply
- Scripts live at `alembic/versions/`

### SQLAlchemy Models
- ORM models live only in `infrastructure/` — never imported in domain or application layers
- Map ORM rows to domain models inside repository implementations
