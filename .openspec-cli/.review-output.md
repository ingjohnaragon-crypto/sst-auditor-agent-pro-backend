**File**: `contracts/fixed_term_deposit.py` (lines 116–133) — **Severity: MAJOR** *(unresolved — 3rd review)*

`accrued_interest` and `days_to_maturity` are still declared as regular `Parameter` objects with `derived=True`. `DerivedParameter` is absent from both the imports and from a `derived_parameters` module-level list. `derived=True` is not a recognised Vault API 4.0 kwarg on `Parameter` — Vault will not route `derived_parameter_hook` calls for these values, silently ignoring the hook in a live environment.

```python

Parameter(name="days_to_maturity",  shape=NumberShape(), level=ParameterLevel.INSTANCE, derived=True, ...)

# REQUIRED — add to imports and replace with:
from contracts_api import DerivedParameter   # add to imports

derived_parameters = [                       # module-level list, after parameters
    DerivedParameter(name="accrued_interest", shape=NumberShape(), display_name="Accrued Interest"),
    DerivedParameter(name="days_to_maturity",  shape=NumberShape(), display_name="Days to Maturity"),
]
```

**File**: `contracts/fixed_term_deposit.py` (lines 84, 199, 381) — **Severity: MAJOR** *(unresolved — 3rd review)*

`maturity_date` is declared as `OptionalShape(shape=DateShape())`. In real Vault, `DateShape` returns a `datetime.date` object from `.latest()`. Calling `.date()` on a `date` raises `AttributeError`. Tests pass only because `make_vault` wraps a `datetime` object in `OptionalValue`, masking the runtime divergence.

```python
# line 199 — WRONG
if maturity_date.date() <= effective_date:   # AttributeError on datetime.date

# line 381 — WRONG
days = max((opt_maturity.value.date() - today).days, 0)  # same
```

Required fix: change `maturity_date` parameter shape from `OptionalShape(shape=DateShape())` to plain `DateShape()`, remove all `.is_set()` / `.value` unwrapping and `.date()` calls in `activation_hook` and `derived_parameter_hook`. Update test fixtures — `FUTURE_MATURITY`, `TODAY_MATURITY`, `PAST_MATURITY` must be `date` objects (e.g. `datetime(2025, 6, 30, tzinfo=UTC).date()`).
